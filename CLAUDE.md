## Commands

```bash
bun install          # install dependencies
bun run dev          # local dev server via wrangler dev
bun run deploy       # deploy to Cloudflare (minified)
bun run cf-typegen   # regenerate worker-configuration.d.ts from wrangler.jsonc bindings
```

## Architecture

This is a **Cloudflare Worker** using the **Hono** framework, bootstrapped with `create-cloudflare`.

- `src/index.ts` — sole entry point; exports a Hono app as the Worker's default export.
- `public/` — static assets served via the `ASSETS` binding (Cloudflare Assets).
- `wrangler.jsonc` — Worker configuration: name, entry, compatibility date, bindings, and feature flags.
- `worker-configuration.d.ts` — auto-generated by `bun run cf-typegen`; do not edit manually. Defines the `CloudflareBindings` interface used to type `c.env` in Hono handlers.

**Bindings** declared in `wrangler.jsonc` must be reflected by re-running `cf-typegen` before TypeScript will accept them. Currently the only binding is `ASSETS: Fetcher`.

**JSX** is configured to use `hono/jsx` (`jsxImportSource` in `tsconfig.json`), so JSX can be used directly in route handlers without importing React.

The `nodejs_compat` compatibility flag is enabled, allowing use of Node.js built-in APIs inside the Worker runtime.
