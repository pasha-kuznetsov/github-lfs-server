# AGENTS.md

## Architecture

**Cloudflare Worker** using **Hono**.

- `src/index.ts` — sole entry point; exports Hono app as Worker's default export.
- `wrangler.jsonc` — symlinked from deploy repo root; don't edit inside `server/`. `wrangler.template.jsonc` is Handlebars template it's rendered from.
- `worker-configuration.d.ts` — symlinked from deploy repo root; auto-generated by `bun run types`. Defines `CloudflareBindings` interface for typing `c.env` in Hono handlers.
- `public/` — static assets served via `ASSETS` binding (Cloudflare Assets).

**Bindings** declared in `wrangler.jsonc` must be reflected by re-running `types` before TypeScript accepts them. Current: `ASSETS` (static assets), `LFS_BUCKET` (R2), `LOCKS` (Durable Object).

**JSX** configured to use `hono/jsx` (`jsxImportSource` in `tsconfig.json`) — use JSX in route handlers without importing React.

## Bun instead of node/npm/npx/pnpm/yarn

- Use `bun <file>`, not `node <file>` or `ts-node <file>`
- Use `bun install`, not `npm install` or `yarn install` or `pnpm install`
- Use `bun run <script>`, not `npm run <script>` or `yarn run <script>` or `pnpm run <script>`
- Use `bunx <package> <command>` instead of `npx <package> <command>`
- Bun automatically loads .env, so don't use dotenv.

BUT: enabled `nodejs_compat` flag uses Node.js built-in APIs in Worker _runtime_.

## Commands

```bash
bun install          # install dependencies
bun run dev          # local dev server via wrangler dev
bun run deploy       # deploy to Cloudflare (minified)
bun run types        # regenerate worker-configuration.d.ts from wrangler.jsonc bindings
bun run test         # run tests via vitest + @cloudflare/vitest-pool-workers (no build step needed)
```

## Testing

Use `vitest` to run tests.

```ts#index.test.ts
import { test, expect } from "vitest";

test("hello world", () => {
  expect(1).toBe(1);
});
```
